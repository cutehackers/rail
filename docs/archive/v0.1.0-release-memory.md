# `v0.1.0` Release Memory

Date: 2026-04-13  
Tag: `v0.1.0`

## What Shipped

`v0.1.0` is the production release of the `v1` core supervisor gate.

Released scope:

- `compose-request`
- `validate-request`
- `run --request ... --project-root ...`
- `execute --artifact ...`
- `route-evaluation`
- sequential `planner -> context_builder -> generator -> executor -> evaluator`
- evaluator-driven bounded corrective loop
- deterministic supervisor routing for the supported `v1` actions
- release-gate automation in local script and CI

## What Did Not Ship

The following remain intentionally outside `v0.1.0`:

- `integrator`
- `apply-user-outcome-feedback`
- `apply-learning-review`
- `apply-hardening-review`
- approved-memory reuse in live runs
- review queue and hardening queue lifecycle operations
- quality-improvement-over-time workflows

These are `v2` items, not hidden `v1` debt.

## Release Meaning

`v0.1.0` means the bounded supervisor gate is release-ready within its declared scope.

It does not mean:

- broader workflow orchestration is complete
- post-pass integration is production-ready
- self-learning is active in the runtime path
- every future task family is already hardened

## Architectural Memory

Keep these boundaries intact unless the product scope changes explicitly:

- the target application repository is external and always supplied via `--project-root`
- the bounded corrective loop is part of `v1`
- the self-learning loop is not part of `v1`
- `pass` semantics terminate at evaluator; post-pass handoff is not implied
- `v2` work must remain isolated from the `v1` runtime path until separately verified
